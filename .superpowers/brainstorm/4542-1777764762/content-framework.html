GOC AI Dashboard 内容框架地图

用于后续接收你上传的产品文档、案例数据、截图素材，并填充成完整 landing page copy。

01 / Hero

核心任务

在 5 秒内说明 GOC 解决什么问题：用统一跨平台归因标准回答广告效果如何优化。

需要补充

产品英文名、主 CTA 文案、是否已有免费试用入口、是否需要预约 Demo 作为次 CTA。

02 / Problem

核心任务

让广告主意识到现有平台归因存在系统性偏差：各平台口径不同、倾向归功于自身渠道、跨平台效果割裂。

需要补充

典型客户遇到的归因冲突例子，例如 Meta、Google、TikTok 同时 claim conversion。

03 / Method

核心任务

把 GOC 的方法讲成一套更可信的归因框架：统一标准、跨设备、长时效、用户转化周期。

需要补充

归因逻辑是否有专有方法名、窗口期范围、支持哪些平台和事件类型。

04 / Capabilities

核心任务

用三大能力承接核心痛点：真实归因、中上层渠道衡量、优化预算决策。

需要补充

每个能力的截图、功能列表、客户能得到的具体指标或报告。

05 / Dashboard Preview

核心任务

把抽象的归因逻辑变成可见界面：渠道贡献、转化路径、周期分析、预算建议。

需要补充

Dashboard 截图或原型、最想展示的 3 个核心图表、是否需要脱敏客户数据。

06 / Proof + CTA

核心任务

建立可信度并推动免费试用：客户 Logo、案例指标、试用能看到的价值、低风险承诺。

需要补充

已有客户行业、可公开指标、试用周期、试用期间是否需要技术支持。
