# GOC 01 — 账号注册页与 GOC 审批门禁设计

**状态：** 已定稿（含前端原型）  
**日期：** 2026-05-05  
**关联：** `2026-05-03-goc-login-saas-vertical-layout-design.md`（登录页信息架构；注册仅对外主路径）

---

## 1. 目标

- 提供自助 **注册申请** 入口，与登录页 「还没有账号？注册」 互通。
- **极简表单（方案 A）**：用户名称、密码、企业名称、邮箱；算术验证码；必选同意条款（服务条款/隐私政策）。
- **GOC 审批门禁**：用户提交后仅为申请；**须经 GOC 审批通过后**，账号/租户方可 **创建成功并允许登录**。
- **登录策略（方案 B）**：审批完成前，**不允许登录**；登录接口应返回可区分状态（如「审核中」），前端映射为明确文案，避免误判为密码错误。
- 注册页 **不包含** 飞书/Google 区块（与既有登录规格一致：OAuth 留在登录页）。

---

## 2. 页面与字段

| 字段 | 控件 | 前端校验（原型） |
|------|------|------------------|
| 用户名称 | text | 必填 |
| 密码 | password | 必填；建议最小长度 8（可与后端策略对齐） |
| 企业名称 | text | 必填 |
| 邮箱 | email | 必填；浏览器/email 基础格式 |
| 验证码 | text + 算式示意 | 必填；原型可与固定算式答案校验 |
| 条款 | checkbox | 必选才可提交 |

---

## 3. 文案与说明（必选）

- **表单区域上方**：醒目标明「提交后为注册申请；**需 GOC 审批通过后方可开通账号并登录**，请留意注册邮箱」。
- **提交成功后**：结果区再次说明已收到申请、审批通过后将开通、审批完成前无法登录；提供「返回登录」链至 `login.html`。

---

## 4. 交互状态

| 状态 | 说明 |
|------|------|
| 默认 | 展示表单 + 审批说明 |
| 校验失败 | 就近提示（原型可用 `alert` 或行内，正式环境建议行内） |
| 提交成功（原型） | 隐藏表单，展示成功说明（无真实后端，本地模拟） |

---

## 5. 验收要点

- `login.html` 「还没有账号？注册」指向 `register.html`；注册页有「已有账号？去登录」指向 `login.html`。
- 页面视觉与 `login.css` 体系一致；侧栏与登录页同构。
- 审批说明与成功态文案符合第 3 节。
- 登录页可加一句弱提示：新注册需审核通过后方可登录（可选，已实现则检查文案）。

---

## 6. 后续（接后端时）

- POST 注册申请 API；重复邮箱/企业策略与错误码映射。
- 登录 API：未通过审批返回独立 `code`，前端展示「审核中」类提示。
- 邮件通知：申请已收到 / 审批通过 / 驳回（ copy 与法务确认）。

---

## 7. 明确不包含（本期原型）

- Google/飞书自助注册、确认密码、邮箱内链验证闭环、真实短信、运营后台审批 UI。
