# GOC01 项目列表与成员管理设计

**日期：** 2026-05-07
**范围：** 设置页重构 — 从单项目配置改为多项目列表管理，增加成员管理功能
**技术栈：** 纯前端静态 demo（HTML + CSS + 原生 JS + localStorage）

---

## 背景与问题

当前设置页（`app.html` 中的 `#panel-settings`）直接展示单个项目的配置表单。新用户注册后进入设置页就是一个项目信息配置界面，没有多项目管理的概念。这不合理——用户应该能管理多个项目。

## 需求摘要

1. 设置页从"单项目配置表单"改为"项目列表"
2. 每个用户最多可创建 5 个项目
3. 项目列表中每个项目卡片有"查看配置"（只读）和"成员管理"两个操作
4. 成员管理为简单的账号列表（添加/移除），demo 不需要真实邀请
5. 新用户首次进入看到空的项目列表 + "新建项目"按钮

---

## 整体架构：设置页内四视图切换

在 `#panel-settings` 内部，通过 JS 切换四个子视图：

```
侧边栏 [设置] 点击
    └── 项目列表视图 (默认入口)
            ├── [新建项目] → 新建项目视图 (复用现有配置表单)
            │       └── 提交成功 → 回到列表视图
            └── 项目卡片操作
                    ├── [查看配置] → 配置详情视图 (只读)
                    │       └── [返回] → 回到列表视图
                    └── [成员管理] → 成员管理视图
                            └── [返回] → 回到列表视图
```

顶栏的项目下拉保留，用于快速切换当前活跃项目（影响仪表盘等页面的数据展示）。现有的内置 demo 项目（`builtin-demo` 等）在项目列表中不显示，只出现在顶栏下拉中。

## 视图 1：项目列表

### 布局

- **顶部标题栏：** "我的项目" + 右侧"新建项目"按钮 + 已用额度提示（如 `2/5`）
- **卡片网格：** 每行 2-3 个卡片，响应式

### 项目卡片内容

每张卡片展示：

| 元素 | 说明 |
|------|------|
| 项目名称 | 如 `apple` |
| 区域 + 产品线 | 如 `us · all` |
| Google 授权状态 | 标签："已连接" 或 "未连接" |
| 「查看配置」按钮 | 进入只读配置详情视图 |
| 「成员管理」按钮 | 进入成员管理视图 |

### 空状态

新用户首次进入时：
- 居中提示文案："您还没有项目，点击上方按钮创建第一个项目"
- "新建项目"按钮正常可用

### 达到上限（5/5）

- "新建项目"按钮变灰（disabled）
- Tooltip 提示："已达到项目数量上限（5/5）"

---

## 视图 2：新建项目

### 入口

项目列表视图中点击"新建项目"按钮。

### 内容

**完整复用现有的项目配置表单**（`#project-config-form`），包括：
- 项目名称（必填）
- 项目区域
- 产品线
- Google 数据源区块（一键授权、GA4、GTM、BigQuery、Google Ads）
- 底部"确定"按钮

### 行为

- 顶部显示面包屑：`我的项目 > 新建项目`，带返回按钮
- 提交成功后：项目存入 localStorage，跳转回项目列表视图
- 取消/返回：回到项目列表视图，不保存

---

## 视图 3：查看配置（只读）

### 入口

项目卡片上的"查看配置"按钮。

### 布局

- **顶部：** 面包屑 `我的项目 > {项目名称}` + 返回按钮
- **内容：** 复用配置表单的字段布局，但所有字段为只读文本展示（非输入框）

### 分区

1. **项目基本信息：** 项目名称、区域、产品线
2. **Google 数据源：** GA4 账号、GTM 账号/Workspace、GTM 容器、BigQuery 数据集、Google Ads 账号 — 每项显示当前值和连接状态

### 不提供编辑功能

此视图纯展示，不可修改。

---

## 视图 4：成员管理

### 入口

项目卡片上的"成员管理"按钮。

### 布局

- **顶部：** 面包屑 `我的项目 > {项目名称} > 成员管理` + 返回按钮
- **添加成员区域：** 输入框（邮箱/账号）+ "添加"按钮，同行排列
- **成员列表表格**

### 成员列表表格

| 列 | 说明 |
|----|------|
| 用户账号 | 邮箱地址 |
| 角色 | "创建者"或"成员" |
| 添加时间 | 格式 `YYYY-MM-DD` |
| 操作 | 移除按钮（创建者不可移除） |

### 行为

- **添加：** 输入邮箱后点击"添加"，直接出现在列表中（demo 不做真实验证）
- **移除：** 点击移除按钮，弹出确认后从列表中删除
- **创建者：** 项目创建时自动添加，角色标记为"创建者"，移除按钮禁用
- **空状态：** 仅显示创建者自己，下方提示"添加其他成员来协作管理此项目"

## 数据结构（localStorage）

沿用现有的 `goc01_dashboard_projects_demo` 键，扩展项目对象：

```js
{
  userProjects: [
    {
      id: 'user-1715000000000',
      label: 'us · apple',
      config: {
        projectName: 'apple',
        region: 'us',
        productLine: 'all',
        googleConnected: true,
        ga4: 'GA4：示例账号 A',
        googleAds: 'Google Ads：示例 MCC',
        bqDataset: '__goc_delegate__',
        gtm: '__goc_delegate__',
        gtmPublicId: '__goc_delegate__',
        gtmDelegateCreate: true,
        gtmPublicDelegateCreate: true,
        bqDelegateCreate: true
      },
      members: [
        { email: 'creator@example.com', role: 'creator', addedAt: '2026-05-07' }
      ]
    }
  ],
  selectedId: 'user-1715000000000'
}
```

**新增字段：** `members` 数组，每个成员对象包含 `email`、`role`（`creator` | `member`）、`addedAt`。
