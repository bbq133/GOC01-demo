# 成员邀请机制 UX 闭环设计

**日期：** 2026-05-14  
**状态：** 已批准，待实施  
**范围：** `app.html`、`register.html`（前端 Demo，localStorage 数据层）

---

## 背景与问题

当前成员管理页有两个邀请入口：

1. **邀请链接**（`register.html?invite=<token>`）：新用户点链接注册后自动加入项目 ✅ 逻辑完整
2. **邮箱直接添加**：管理员填邮箱点「添加」→ 该邮箱立即以 `普通成员` 状态写入成员表，无任何中间状态、无反馈 ❌ 断层

核心断层：**"添加"是静默的** —— 对方不知道被加入；成员表里的人不一定有真实账号；演示时观众看不到邀请流程。

此外，`alert()` 的使用破坏了 Demo 整体质感。

---

## 设计目标

1. 让"邮箱添加"呈现完整的邀请状态机：**待加入 → 接受 → 已加入**
2. 通过邀请链接注册的成员直接为 `active`，无需 pending
3. 用内联 toast/按钮状态反馈替换所有 `alert()`
4. 数据改动向后兼容，不破坏已有数据

---

## 数据模型

`members[]` 数组中每个成员对象新增可选字段：

```js
{
  email: "alice@example.com",
  role: "member",     // 'admin' | 'member'
  status: "pending",  // 'active' | 'pending'  ← 新增
  addedAt: "2026-05-14"
}
```

- **`active`**：已正式加入（通过邀请链接注册，或旧数据兼容默认值）
- **`pending`**：邮箱已添加，尚未"接受邀请"
- 无 `status` 字段的历史记录默认视为 `active`
- 管理员始终为 `active`

## UI 变更

### 成员表格行

| 状态 | 角色列 badge | 操作列（管理员视角） |
|------|-------------|---------------------|
| `active` 管理员 | `管理员` / `管理员（项目负责人）` 蓝紫色 | 「当前帐号」或 `—` |
| `active` 普通成员 | `普通成员` 灰色 | 「设为管理员」「移除」 |
| `pending` 任意角色 | `待加入` 黄色 | 「模拟接受邀请」「撤销邀请」 |

### 邮箱添加流程

1. 管理员填邮箱 → 点「添加」（或回车）
2. 若邮箱已存在（任意状态） → 输入框下方红色提示「该成员已存在」
3. 否则：以 `status: 'pending'` 写入 localStorage，渲染到成员表为「待加入」黄色 badge
4. 输入框下方出现绿色小提示条「已发送邀请：xxx@xxx.com」，2 秒后消失

### 模拟接受邀请

- 管理员点「模拟接受邀请」→ 该成员 `status` 改为 `active`
- badge 从「待加入」变为「普通成员」
- 操作列变为「设为管理员」「移除」

### 撤销邀请

- 管理员点「撤销邀请」→ 从 `members[]` 中移除该记录
- 表格立即更新，无弹窗

### 通过邀请链接注册

- `register.html` 写入成员时直接设 `status: 'active'`（注册即加入）

### Toast 替换 alert()

| 原 alert | 新反馈方式 |
|----------|-----------|
| 复制邀请链接成功 | 按钮文字变「已复制 ✓」，2 秒后恢复「复制链接」 |
| 添加成员成功 | 输入框下绿色提示条，2 秒消失 |
| 添加成员已存在 | 输入框下红色提示条，不消失直到用户重新输入 |
| 移除/撤销 | 无提示，表格直接更新 |

## 实施范围

**`app.html` 修改：**
- `addMember()` 函数：写入时加 `status: 'pending'`，成功后显示绿色 toast
- `renderMemberTable()` 函数：读取 `status` 字段，渲染对应 badge 和操作按钮
- 新增 `acceptInvite(email)` 函数：将指定成员 `status` 改为 `active`
- `removeMember()` 函数：兼容撤销 pending 邀请（逻辑相同，可复用）
- 复制链接按钮：改为文字切换反馈
- 新增 `.invite-toast` 内联提示 DOM 及相关 CSS

**`register.html` 修改：**
- 注册成功写入成员时，加上 `status: 'active'`
