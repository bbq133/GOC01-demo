# 项目配置 · Google 一键授权与分项重新授权（前端原型）

**状态：** 已实现（静态原型）  
**日期：** 2026-05-05  
**范围：** 仅 `site/app.html` + `site/app.css`，无后端、无真实 OAuth。

---

## 1. 目标

- 在「项目配置」表单中，为 GA4 / Google Ads / GTM / GTM PublicId 增加**同一 Google 邮箱**语境下的 **一键授权**入口与说明。
- 授权成功后**不自动选中**任一下拉项，用户需逐项手动选择（决策 **B**）。
- 各子项保留 **单独重新授权**：标签与「重新授权」同一行，链接在右（决策 **A**）。

## 2. 布局（方案一）

- 产品线字段下方增加顶部分割线与区块 **「Google 数据源」**（`.form-google-block`）。
- 面板（`.google-auth-panel`）含：标题、说明文案、按钮「使用 Google 账号一键授权」、`role="status"` 状态行。
- 其下四个 `form-group`：`.label-row` 内 `label` + `button.link-reauth`（`type="button"`），下方为 `select.google-select`。

## 3. 交互（原型脚本）

- **未连接**：四个下拉 `disabled`，占位文案为「请先完成上方 Google 一键授权后再选择」；状态行为「尚未连接 Google 账号。」
- **首次一键授权**：解除 `disabled`，占位恢复为各字段原有业务说明；`selectedIndex = 0`（占位项），**不**自动选具体账号；状态行显示模拟邮箱。
- **再次一键授权**（已连接）：**不改变**已选下拉值，仅更新状态为「已刷新 Google 授权（原型）…」。
- **重新授权**：已连接时状态行显示「已模拟重新发起「…」授权请求」；未连接时提示先完成一键授权。

## 4. 样式要点

- 一键按钮为 **outline 主色**，与表单底部实心「确定」区分。
- 禁用下拉略降低不透明度（`.is-locked` / `:disabled`）。
- `link-reauth`：蓝字下划线，`focus-visible` 描边。

## 5. 接入真 OAuth 时（备忘）

- 一键与分项重新授权可共用 refresh token 逻辑；状态行展示真实授权邮箱与过期提示；错误提示贴近面板或对应 `label-row`。
