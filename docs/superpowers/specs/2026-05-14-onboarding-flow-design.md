# GOC01 新项目创建向导与进度展示设计规范 (Design Spec)

## 1. 概述 (Overview)
本文档定义了 GOC01 产品中“新项目创建”的前端交互流程及“项目准备中”状态的视觉与逻辑设计。该设计基于细分向导（Step-by-step Onboarding）模式，旨在降低用户的认知负荷，并清晰展示异步长耗时任务（数据积累与模型训练）的进度。

## 2. 第一部分：创建向导流程 (Onboarding Flow)

### 2.1 整体交互框架
- **入口**：项目列表页点击“新建项目”按钮。
- **形式**：全屏或大尺寸的模态弹窗（Modal/Drawer）。
- **导航**：顶部带有步骤指示器（Step Indicator）；底部提供“上一步”、“下一步”按钮。当前步骤必填项完成后，“下一步”才可点击。

### 2.2 步骤拆解

#### Step 1: 基础信息 (Basic Info)
- **输入项**：
  - 项目名称（必填，后端校验是否重复）
  - 项目区域（必填，默认 `all`）
  - 产品线（必填，默认 `all`）

#### Step 2: 核心数据源授权 (Core Authorization)
- **说明文案**：“系统需要 GA4 和 BigQuery 权限以进行数据分析。我们将自动为您检测或创建所需的数据集。”
- **操作**：“使用 Google 账号一键授权”主按钮。
- **状态反馈**：授权回调后，展示 GA4 和 BigQuery 的授权状态（成功/失败）。失败则提供“重新授权”按钮。
- **逻辑判断**：点击“下一步”时，前端展示 Loading（“正在初始化数据集...”），后端执行 `BqLinkCheck`，判断是否需要代为创建 GCP 项目和 BQ 数据集（PIGOC）。

#### Step 3: 附加配置 (Optional Setup)
- **Google Ads 授权**：
  - 单选：“您的品牌是否投放 Google Ads？”（是/否）。
  - 选“是”则展示授权按钮，授权后后端自动绑定 GA 与 Ads。
- **GTM 配置**：
  - 单选：“是否需要配置 Google Tag Manager 自动埋点？”（是/否）。
  - 选“是”则展示授权按钮，授权后后端异步导入 Tag。
- **完成操作**：点击“提交并创建项目”。

## 3. 第二部分：“项目准备中”状态页 (Preparation Status Page)

### 3.1 页面整体布局
当项目创建成功但数据/模型未准备就绪时，用户点击项目卡片将进入此专属状态页。
- **页面标题**：`[项目名称] - 数据初始化中`
- **顶部提示 (Banner)**：“🚀 您的项目正在努力准备中。数据积累和 AI 模型训练通常需要 7 天左右，完成后我们将通过邮件通知您。”
- **核心视觉**：页面中央的垂直时间轴（Vertical Stepper）。
- **菜单限制**：左侧导航栏的“AI 数据分析”、“运营数据”等菜单处于禁用（Disabled）状态。

### 3.2 时间轴节点设计

| 节点 | 阶段名称 | 状态 | 动态文案与 UI 细节 |
| :--- | :--- | :--- | :--- |
| 1 | 项目与数据集初始化 | ✅ 已完成 | “项目配置已保存，BigQuery 数据集 (PIGOC) 初始化成功。” |
| 2 | 数据同步与积累 | ⏳ 进行中 | - 若后端代为创建 events 表：“正在等待数据生成（预计明日开始累积）”<br>- 若正在同步历史数据：“已同步 X / 7 天数据”（附带进度条，Max=7, Value=X） |
| 3 | AI 模型训练 | 待开始 / 进行中 | “启动 2.0 / 3.0 算法双轨训练，生成浓度预测模型。”<br>*(兜底逻辑提示：若触发兜底，可小字提示“正在自动调整参数进行深度训练...”)* |
| 4 | 生成浓度报告 | 待开始 | “模型训练完成，即将为您呈现完整数据看板。” |

### 3.3 异常与兜底交互
- **无限等待状态**：若模型训练彻底失败（触发兜底后仍失败），进入“无限等待，继续积累数据”状态。
- **UI 更新**：节点 2 和节点 3 的状态文案更新为：“模型需要更多数据进行学习，正在继续累积数据 (已累积 X 天)”。
